System

show system uptime

show system uptime

What it does: Displays how long the system has been running.

When to use it: To check for recent reboots.

show version

show version

What it does: Displays JunOS version and system info.

When to use it: To identify the software version and hardware.

Interface

show interfaces terse

show interfaces terse

What it does: Quick interface status overview.

When to use it: A primary command to check interface status and IP configuration.

Routing

show route

show route

What it does: Displays routing table entries.

When to use it: To inspect the device's routing table and verify route paths.

show route protocol bgp

show route protocol bgp

What it does: Verifies BGP learned routes.

When to use it: To see routes learned specifically via the BGP routing protocol.

VPN

show security ike security-associations

show security ike security-associations

What it does: Checks IKE Phase 1 VPN status.

When to use it: To verify if the IKE security association (Phase 1) is established.

show security ipsec security-associations

show security ipsec security-associations

What it does: Displays IPsec Phase 2 tunnels.

When to use it: To verify if the IPsec security association (Phase 2) is established and actively passing traffic.

Configuration

configure

configure

What it does: Enters configuration mode.

When to use it: To make any changes to the device's configuration.

set interface ge-0/0/0

set interfaces ge-0/0/0 unit 0 family inet address 192.168.1.1/24

What it does: Sets an IP address on a logical interface.

When to use it: To configure an interface IP address.

set routing-options static

set routing-options static route 0.0.0.0/0 next-hop 192.168.1.254

What it does: Creates a static route. This example creates a default route.

When to use it: To manually define a path for traffic.

commit

commit

What it does: Validates and applies the candidate configuration.

When to use it: After making configuration changes, you must commit them to make them active.

Troubleshooting

show route

show route

What it does: Displays the routing table.

When to use it: A primary command to troubleshoot routing issues.

show log messages

show log messages

What it does: Reviews system logs for troubleshooting.

When to use it: To check for errors, events, and traffic logs.

monitor traffic interface <interface>

monitor traffic interface ge-0/0/0.0 detail

What it does: Live packet capture on interface.

When to use it: For real-time packet inspection on a specific interface to diagnose network issues.

ping <ip> rapid count 5

ping 8.8.8.8 rapid count 5

What it does: Fast connectivity testing.

When to use it: For basic connectivity testing with a quick burst of pings.

traceroute <ip>

traceroute 8.8.8.8

What it does: Route path analysis.

When to use it: To identify routing loops or points of failure.
