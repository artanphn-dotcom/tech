Compute Engine (List/Connect)

gcloud compute instances list

gcloud compute instances list

What it does: Confirms VM status and IP assignments.

When to use it: To get a quick overview of your virtual machines and their statuses.

gcloud compute ssh

gcloud compute ssh [INSTANCE_NAME] --zone [ZONE]

What it does: Connects to a Compute Engine instance using SSH.

When to use it: To securely access the command line of a VM.

Network (List/View)

gcloud compute networks list

gcloud compute networks list

What it does: Lists available VPC networks.

When to use it: To review configured Virtual Private Cloud networks.

gcloud compute routes list

gcloud compute routes list

What it does: Displays routing table for a VPC.

When to use it: To verify network routes and troubleshoot traffic flow issues.

gcloud compute firewall-rules list

gcloud compute firewall-rules list

What it does: Shows firewall rules and priorities.

When to use it: To examine configured firewall policies and ensure desired traffic control.

gcloud compute networks subnets list

gcloud compute networks subnets list

What it does: Lists subnet ranges and regions.

When to use it: To review subnet configurations within your VPC networks.

Kubernetes Engine

gcloud container clusters list

gcloud container clusters list

What it does: Lists all GKE clusters in the project.

When to use it: To see your Kubernetes clusters and their locations.

gcloud container clusters get-credentials

gcloud container clusters get-credentials [CLUSTER_NAME] --zone [ZONE]

What it does: Configures `kubectl` with credentials to access your GKE cluster.

When to use it: The first step to interacting with your cluster using `kubectl`.

Cloud Storage

gcloud storage ls

gcloud storage ls gs://[BUCKET_NAME]

What it does: Lists the contents of a Cloud Storage bucket.

When to use it: To view the objects stored in a bucket.

gcloud storage cp

gcloud storage cp [LOCAL_FILE] gs://[BUCKET_NAME]/

What it does: Copies files between your local machine and Cloud Storage.

When to use it: To upload or download files from a storage bucket.

VPN

gcloud compute vpn-tunnels list

gcloud compute vpn-tunnels list

What it does: Checks VPN tunnel status.

When to use it: To quickly ascertain the operational state of your Cloud VPN tunnels.

gcloud compute routers get-status <router>

gcloud compute routers get-status my-cloud-router

What it does: Verifies Cloud Router and BGP state.

When to use it: To check the status of BGP sessions on a Cloud Router, particularly for HA VPNs.

gcloud compute routers describe <router>

gcloud compute routers describe my-cloud-router

What it does: Displays BGP peers and ASN details.

When to use it: To get detailed configuration and status of a Cloud Router and its BGP peers.

gcloud compute vpn-gateways describe <name>

gcloud compute vpn-gateways describe my-ha-vpn-gateway

What it does: Displays HA VPN gateway details.

When to use it: To review the configuration and status of a Highly Available VPN gateway.

Configuration

gcloud compute instances create

gcloud compute instances create my-vm --machine-type=e2-medium --image-family=ubuntu-2004-lts --image-project=ubuntu-os-cloud

What it does: Creates a new Google Compute Engine virtual machine.

When to use it: When you need to provision a new server in GCP.

Troubleshooting

gcloud logging read

gcloud logging read "resource.type=gce_instance AND resource.labels.instance_id=[INSTANCE_ID]" --limit=50

What it does: Reads logs for firewall, VPN, or network issues.

When to use it: To view logs for a specific resource, such as a VM instance, to diagnose issues.
