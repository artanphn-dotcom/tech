Core Commands

terraform init

terraform init

What it does: Initializes a new or existing Terraform configuration, downloading provider plugins and modules.

When to use it: The first command to run after writing a new Terraform configuration or cloning an existing one.

terraform validate

terraform validate

What it does: Checks whether a configuration is syntactically valid and internally consistent.

When to use it: As a quick check for errors before running a plan or apply.

terraform fmt

terraform fmt -recursive

What it does: Rewrites Terraform configuration files to a canonical format and style.

When to use it: To ensure consistent formatting across all your configuration files.

terraform plan

terraform plan

What it does: Creates an execution plan, which lets you preview the changes that Terraform plans to make.

When to use it: Before applying changes, to ensure they are what you expect.

terraform apply

terraform apply

What it does: Applies the changes required to reach the desired state of the configuration.

When to use it: To create, update, or delete infrastructure resources.

terraform destroy

terraform destroy

What it does: Destroys all the resources managed by the current Terraform configuration.

When to use it: To tear down infrastructure created by Terraform. Use with caution.

State Management

terraform state list

terraform state list

What it does: Lists all the resources that are tracked in the current Terraform state file.

When to use it: To get a quick overview of what infrastructure Terraform is managing.

terraform state show

terraform state show 'module.vpc.aws_vpc.this'

What it does: Shows the attributes of a single resource in the state file.

When to use it: To inspect the details of a specific resource managed by Terraform.

terraform state rm

terraform state rm 'aws_instance.my_instance'

What it does: Removes a resource from the Terraform state, without destroying the actual infrastructure.

When to use it: To "unmanage" a resource that was previously managed by Terraform.

Troubleshooting

TF_LOG=TRACE terraform plan

export TF_LOG=TRACE
terraform plan

What it does: Enables detailed logging for Terraform operations by setting the `TF_LOG` environment variable.

When to use it: When you need to debug issues with providers, state, or the Terraform core process. Other log levels include `DEBUG`, `INFO`, `WARN`, `ERROR`.
