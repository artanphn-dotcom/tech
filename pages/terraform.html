Core Commands

terraform init

terraform init

What it does: Initializes a new or existing Terraform configuration, downloading provider plugins and modules.

terraform validate

terraform validate

What it does: Checks whether a configuration is syntactically valid and internally consistent.

terraform fmt

terraform fmt -recursive

What it does: Rewrites Terraform configuration files to a canonical format and style.

terraform plan

terraform plan -out=tfplan

What it does: Creates an execution plan, which lets you preview the changes Terraform plans to make, saving it to a file.

terraform apply

terraform apply "tfplan"

What it does: Applies the changes required to reach the desired state of the configuration, using a saved plan.

terraform destroy

terraform destroy

What it does: Destroys all the resources managed by the current Terraform configuration.

terraform console

terraform console

What it does: Provides an interactive console for evaluating expressions and interacting with the Terraform state.

terraform import

terraform import aws_instance.example i-1234567890abcdef0

What it does: Imports existing infrastructure into Terraform state.

terraform output

terraform output instance_ip

What it does: Extracts the value of an output variable from the state file.

terraform workspace new

terraform workspace new development

What it does: Creates a new workspace.

terraform workspace select

terraform workspace select production

What it does: Selects an existing workspace.

terraform graph

terraform graph | dot -Tpng > graph.png

What it does: Generates a visual graph of Terraform resources.

terraform taint

terraform taint aws_instance.example

What it does: Explicitly marks a resource as tainted, forcing it to be replaced on the next apply.

terraform untaint

terraform untaint aws_instance.example

What it does: Removes the tainted status from a resource.

terraform show

terraform show

What it does: Shows the current state or a plan file in a human-readable format.

terraform providers

terraform providers

What it does: Prints a tree of the providers used in the current configuration.

terraform version

terraform version

What it does: Prints the Terraform version and provider versions.

terraform get

terraform get -update

What it does: Downloads and updates modules used in the configuration.

terraform login

terraform login

What it does: Authenticates the Terraform CLI with Terraform Cloud/Enterprise.

terraform logout

terraform logout

What it does: Logs out of Terraform Cloud/Enterprise.

State Management

terraform state list

terraform state list

What it does: Lists all the resources that are tracked in the current Terraform state file.

terraform state show

terraform state show 'module.vpc.aws_vpc.this'

What it does: Shows the attributes of a single resource in the state file.

terraform state rm

terraform state rm 'aws_instance.my_instance'

What it does: Removes a resource from the Terraform state, without destroying the actual infrastructure.

terraform state mv

terraform state mv 'aws_instance.old_name' 'aws_instance.new_name'

What it does: Moves an item in the Terraform state, e.g., to rename a resource.

terraform state pull

terraform state pull > current_state.tfstate

What it does: Downloads the current remote state to a local file.

terraform state push

terraform state push updated_state.tfstate

What it does: Uploads a local state file to a remote backend (use with extreme caution).

terraform state replace-provider

terraform state replace-provider "registry.terraform.io/-/aws" "registry.terraform.io/hashicorp/aws"

What it does: Replaces the provider address in the state file, useful after provider refactoring.

terraform state untaint

terraform state untaint aws_instance.tainted_instance

What it does: Removes a tainted mark from a resource in the state.

terraform state list -id=resource_id

terraform state list -id=i-12345abcdef

What it does: Lists resources by their cloud provider ID.

terraform graph -draw-cycles

terraform graph -draw-cycles

What it does: Visualizes resource dependencies, highlighting any dependency cycles.

terraform plan -detailed-exitcode

terraform plan -detailed-exitcode

What it does: Returns a specific exit code indicating whether changes are pending.

terraform apply -auto-approve

terraform apply -auto-approve

What it does: Applies changes without requiring manual confirmation.

terraform output -json

terraform output -json

What it does: Displays all output variables in JSON format.

terraform show -json

terraform show -json

What it does: Displays the state or plan file in JSON format.

terraform refresh

terraform refresh

What it does: Updates the state file by querying the real infrastructure for changes.

terraform console 'aws_instance.example.public_ip'

terraform console 'aws_instance.example.public_ip'

What it does: Evaluates a single expression from the console.

terraform providers schema -json

terraform providers schema -json

What it does: Displays the schema for all configured providers in JSON.

terraform login app.terraform.io

terraform login app.terraform.io

What it does: Authenticates to a specific Terraform Cloud/Enterprise instance.

terraform workspace list

terraform workspace list

What it does: Lists all available workspaces.

terraform workspace show

terraform workspace show

What it does: Displays the name of the current workspace.

Troubleshooting

TF_LOG=TRACE terraform plan

export TF_LOG=TRACE
terraform plan

What it does: Enables detailed logging for Terraform operations by setting the `TF_LOG` environment variable. Other log levels include `DEBUG`, `INFO`, `WARN`, `ERROR`.

terraform validate

terraform validate

What it does: Checks whether a configuration is syntactically valid and internally consistent, catching syntax errors and basic validation failures.

terraform graph

terraform graph | dot -Tpng > dependency_graph.png

What it does: Generates a visual graph of Terraform resources and their dependencies, invaluable for understanding complex configurations and debugging dependency issues.

terraform console

terraform console

What it does: Provides an interactive console for evaluating expressions and inspecting the state, allowing you to test configurations or check resource attributes directly.

terraform refresh

terraform refresh

What it does: Updates the state file by reconciling it with the real-world infrastructure, useful for detecting manual changes or discrepancies.

terraform plan -destroy

terraform plan -destroy

What it does: Generates a plan that shows all resources that would be destroyed, without actually destroying them. Useful for double-checking `terraform destroy` intentions.

terraform state pull

terraform state pull > local.tfstate

What it does: Downloads the latest state file from the configured backend to examine it locally for errors or inconsistencies.

terraform show -json

terraform show -json > state_details.json

What it does: Displays the state file in a machine-readable JSON format, useful for programmatic analysis or detailed inspection.

terraform providers schema -json

terraform providers schema -json

What it does: Displays the schema for all configured providers and resources in JSON, helpful for understanding available arguments and debugging configuration errors.

terraform taint

terraform taint aws_instance.problematic_server

What it does: Manually marks a resource as "tainted," forcing Terraform to replace it on the next `apply`. Useful for recovering from failed resource updates.

terraform graph -type=plan -plan=tfplan

terraform graph -type=plan -plan=tfplan | dot -Tsvg > plan_graph.svg

What it does: Visualizes the changes proposed in a saved plan file, helping to understand the execution order and dependencies of pending operations.

terraform apply -target=aws_instance.specific_instance

terraform apply -target=aws_instance.specific_instance

What it does: Applies changes only to a specific resource, useful for debugging issues with a single resource without affecting others.

terraform state replace-provider

terraform state replace-provider "registry.terraform.io/hashicorp/aws" "registry.terraform.io/custom/aws"

What it does: Modifies the provider address recorded in the state file. Crucial when a provider has changed its upstream location or namespace.

terraform output -raw instance_ip

terraform output -raw instance_ip

What it does: Retrieves the raw output value of a single output variable, useful for scripting.

terraform debug

terraform debug

What it does: Opens a debug console to inspect execution during complex operations (e.g., provider interactions).

TF_CLI_ARGS="-json" terraform plan

TF_CLI_ARGS="-json" terraform plan

What it does: Forces Terraform CLI output into JSON format, useful for programmatic parsing of plans or outputs.

TF_INPUT=0 terraform apply

TF_INPUT=0 terraform apply

What it does: Disables interactive prompts during `terraform apply`, making it non-interactive for automation. (Not `auto-approve`)

terraform graph -type=resource

terraform graph -type=resource

What it does: Generates a graph showing only resource dependencies, simplifying complex infrastructure views.

terraform state get aws_instance.example

terraform state get aws_instance.example

What it does: Retrieves the full attribute values of a specific resource from the state file.

terraform plan -var="region=us-east-1"

terraform plan -var="region=us-east-1"

What it does: Passes a variable value directly via the command line, useful for testing different configurations.
