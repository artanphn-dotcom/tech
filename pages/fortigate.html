FortiGate Commands

System

get system status

get system status

What it does: Displays firmware, uptime, and system health.

When to use it: A primary command for identifying the unit and its operational state.

Interface

get system interface

get system interface physical

What it does: Displays the configuration and status of physical interfaces.

When to use it: To check interface settings, IP addresses, and link status.

Routing

get router info routing-table all

get router info routing-table all

What it does: Shows the complete routing table, including static, connected, and dynamic routes.

When to use it: To verify route paths and troubleshoot routing issues.

VPN

diagnose vpn ike gateway list

diagnose vpn ike gateway list

What it does: Checks IKE Phase 1 VPN status and details.

When to use it: To verify if the IKE security association (Phase 1) is established.

diagnose vpn tunnel list

diagnose vpn tunnel list

What it does: Verifies IPsec tunnel Phase 2 state and traffic selectors.

When to use it: To check if the IPsec security association (Phase 2) is established and actively passing traffic.

Policy

show firewall policy

show firewall policy

What it does: Displays the configured firewall policies.

When to use it: To review the security rulebase.

Configuration Mode

config system interface

config system interface
    edit "port1"
        set ip 192.168.1.99 255.255.255.0
    next
end

What it does: Enters the configuration context for a specific interface to set its IP address.

When to use it: For basic network interface configuration.

config router static

config router static
    edit 1
        set gateway 192.168.1.1
        set device "port1"
    next
end

What it does: Configures a static route.

When to use it: To direct traffic for a specific destination to a next-hop gateway.

Troubleshooting

diagnose debug flow

diagnose debug flow filter addr [ip]
diagnose debug flow show function-name enable
diagnose debug flow trace start 100
diagnose debug enable

What it does: A powerful tool for tracing the path of a packet as it is processed by the FortiGate.

When to use it: Essential for in-depth troubleshooting of complex traffic issues.

execute ping <ip>

execute ping 8.8.8.8

What it does: Basic connectivity testing from firewall.

When to use it: To test basic network connectivity from the FortiGate itself.

execute traceroute <ip>

execute traceroute 8.8.8.8

What it does: Traces packet path for routing issues.

When to use it: To identify routing loops or points of failure in a network path.

diagnose debug application ike -1

diagnose debug application ike -1

What it does: Enables IKE debug for VPN troubleshooting.

When to use it: For in-depth troubleshooting of IPsec VPN Phase 1 negotiations.

diagnose debug enable

diagnose debug enable

What it does: Starts debug output (must be disabled after use).

When to use it: To start viewing debug output after enabling specific debug flags.

diagnose sys session list

diagnose sys session list

What it does: Displays active firewall sessions.

When to use it: To see live traffic flows and verify policy matching.

diagnose sniffer packet any "host <ip>" 4

diagnose sniffer packet any "host 10.0.0.1" 4

What it does: Packet capture directly on FortiGate.

When to use it: For deep packet-level troubleshooting to see actual traffic hitting the firewall.
