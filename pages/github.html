Git Commands

← Back to Home

Repository

git init

git init

What it does: Creates a new Git repository in the current directory.

git clone

git clone [URL]

What it does: Clones a repository into a new directory.

git add

git add .

What it does: Stages changes for the next commit. `.` stages all changes in the current directory.

git commit

git commit -m "Your commit message"

What it does: Records a snapshot of the staged changes to the project's history.

git status

git status

What it does: Displays the state of the working directory and the staging area.

git diff

git diff

What it does: Shows changes between the working directory and the staging area, or between commits.

git log

git log --oneline --graph --decorate

What it does: Shows the commit history with various formatting options.

git show

git show [COMMIT_HASH]

What it does: Displays various types of objects (commits, tags, trees, blobs).

git reset

git reset --hard HEAD~1

What it does: Resets the current HEAD to a specified state, discarding changes.

git revert

git revert [COMMIT_HASH]

What it does: Creates a new commit that undoes the changes from a previous commit.

git clean

git clean -fd

What it does: Removes untracked files from the working directory.

git mv

git mv oldname.txt newname.txt

What it does: Moves or renames a file within the Git repository.

git rm

git rm filename.txt

What it does: Removes a file from the working directory and stages the deletion.

git stash

git stash save "Work in progress"

What it does: Temporarily saves changes that are not ready to be committed.

git restore

git restore file.txt

What it does: Discards changes in the working directory or unstages changes.

git config --list

git config --list

What it does: Lists all Git configuration settings.

git help

git help clone

What it does: Displays help information for a specific Git command.

git reflog

git reflog

What it does: Shows a log of all actions that modified the HEAD of the repository.

git remote show origin

git remote show origin

What it does: Displays detailed information about the 'origin' remote repository.

git pull --rebase

git pull --rebase

What it does: Fetches and rebases changes from the remote, avoiding merge commits.

Branching

git branch

git branch -a

What it does: Lists all local and remote branches.

git checkout

git checkout feature/new-feature

What it does: Switches to a different branch or restores working tree files.

git switch

git switch -c new-feature-branch

What it does: A more modern and safer way to switch branches, especially when creating new ones.

git merge

git merge feature/new-feature

What it does: Integrates changes from one branch into another.

git rebase

git rebase master

What it does: Reapplies commits on top of another base tip, often used to linearize history.

git cherry-pick

git cherry-pick [COMMIT_HASH]

What it does: Applies a specific commit from one branch to another.

git branch -d

git branch -d old-branch

What it does: Deletes a local branch (only if it's been merged).

git branch -D

git branch -D stubborn-branch

What it does: Forcefully deletes a local branch, even if unmerged.

git push origin --delete <branch-name>

git push origin --delete old-remote-branch

What it does: Deletes a remote branch.

git tag

git tag -a v1.0 -m "Version 1.0 release"

What it does: Creates a tag to mark specific points in history as important.

git push --tags

git push --tags

What it does: Pushes all local tags to the remote repository.

git branch --set-upstream-to=origin/main

git branch --set-upstream-to=origin/main

What it does: Sets the upstream branch for the current local branch.

git merge --abort

git merge --abort

What it does: Aborts the current merge operation if there are conflicts.

git rebase --abort

git rebase --abort

What it does: Aborts the current rebase operation.

git bisect start

git bisect start
git bisect bad
git bisect good [GOOD_COMMIT_HASH]

What it does: Helps find the commit that introduced a bug by performing a binary search.

git log --graph

git log --graph --oneline --all

What it does: Visualizes the commit history as a graph.

git shortlog

git shortlog -sn

What it does: Summarizes `git log` output, showing commits per author.

git stash pop

git stash pop

What it does: Applies the most recently stashed changes and then deletes the stash entry.

git stash apply

git stash apply

What it does: Applies the most recently stashed changes, but keeps the stash entry.

git branch --merged

git branch --merged

What it does: Lists branches that have been merged into the current branch.

Remote

git remote

git remote -v

What it does: Lists remote repositories and their URLs.

git fetch

git fetch origin

What it does: Downloads objects and refs from another repository without merging.

git pull

git pull

What it does: Fetches changes from the remote repository and merges them into the current branch.

git push

git push origin [branch-name]

What it does: Pushes your committed changes to a remote repository.

git remote add origin

git remote add origin [URL]

What it does: Adds a new remote repository to your Git project.

git remote set-url origin

git remote set-url origin [NEW_URL]

What it does: Changes the URL for an existing remote repository.

git remote rm origin

git remote rm origin

What it does: Removes a remote repository.

git remote show origin

git remote show origin

What it does: Displays detailed information about the 'origin' remote, including branches and URLs.

git remote prune origin

git remote prune origin

What it does: Deletes any remote-tracking branches that no longer exist on the remote.

git push --force-with-lease

git push --force-with-lease origin feature/branch

What it does: Safely force pushes, ensuring you don't overwrite others' work.

git push -u origin <branch-name>

git push -u origin feature/new-feature

What it does: Pushes the branch and sets it as the upstream tracking branch.

git fetch --all

git fetch --all

What it does: Fetches from all configured remotes.

git pull origin <branch>

git pull origin main

What it does: Pulls from a specific remote and branch.

git config remote.origin.url

git config remote.origin.url

What it does: Displays the URL of the 'origin' remote.

git remote rename <old-name> <new-name>

git remote rename origin upstream

What it does: Renames a remote.

git config --global url."https://github.com/".insteadOf github:

git config --global url."https://github.com/".insteadOf github:

What it does: Configures Git to rewrite `github:` URLs to `https://github.com/`.

git remote update

git remote update

What it does: Fetches updates from all tracked remote branches.

git ls-remote

git ls-remote origin

What it does: Lists references (heads and tags) in a remote repository.

git submodule add <repo-url> <path>

git submodule add https://github.com/user/project.git external/project

What it does: Adds a new submodule to the repository.

git submodule update --init --recursive

git submodule update --init --recursive

What it does: Initializes and updates all submodules.

Configuration

git config --global user.name

git config --global user.name "Your Name"

What it does: Sets the name that will be associated with your commits globally.

git config --global user.email

git config --global user.email "your.email@example.com"

What it does: Sets the email that will be associated with your commits globally.

git config --list

git config --list

What it does: Lists all Git configuration settings (global, system, and local).

git config --global core.editor

git config --global core.editor "code --wait"

What it does: Sets the default text editor for Git to use (e.g., for commit messages).

git config --global alias.co

git config --global alias.co checkout

What it does: Creates a shortcut alias for a Git command (e.g., `git co` instead of `git checkout`).

git config --global init.defaultBranch

git config --global init.defaultBranch main

What it does: Sets the default branch name for new repositories (e.g., `main` instead of `master`).

git config --global color.ui

git config --global color.ui auto

What it does: Enables colorful Git output in the terminal.

git config --local user.name

git config --local user.name "Project Specific Name"

What it does: Sets a user name for the current repository only, overriding the global setting.

git config --system core.pager

git config --system core.pager "less -FSRX"

What it does: Configures the pager for Git output at the system level.

git config --global push.default

git config --global push.default simple

What it does: Sets the default behavior for `git push`.

git config --global pull.rebase

git config --global pull.rebase true

What it does: Configures `git pull` to use rebase instead of merge by default.

git config --global credential.helper

git config --global credential.helper store

What it does: Configures Git to store credentials, avoiding repeated password prompts.

git config --global core.autocrlf

git config --global core.autocrlf input

What it does: Configures how Git handles line endings to prevent cross-platform issues.

git config --global --edit

git config --global --edit

What it does: Opens the global Git configuration file in your default editor for direct editing.

git config --local --unset-all alias.co

git config --local --unset-all alias.co

What it does: Removes a specific alias from the local configuration.

git config --show-origin user.name

git config --show-origin user.name

What it does: Shows the value of a configuration and the file it was read from.

git config --get-regexp "color.*"

git config --get-regexp "color.*"

What it does: Displays all configuration keys matching a regular expression.

git config --global url."https://".insteadOf git://

git config --global url."https://".insteadOf git://

What it does: Configures Git to use HTTPS for all `git://` URLs.

git config --global core.fsmonitor

git config --global core.fsmonitor true

What it does: Enables filesystem monitor for faster `git status` on large repositories.

git config --global blame.ignoreRevsFile

git config --global blame.ignoreRevsFile .git-blame-ignore-revs

What it does: Configures Git to ignore specific revisions when using `git blame`.

Troubleshooting

git status

git status

What it does: Displays the state of the working directory and the staging area.

git log

git log --oneline --graph --decorate --all

What it does: Shows the commit history with various formatting, including all branches.

git diff

git diff HEAD~1

What it does: Shows changes between the working directory and a previous commit.

git reflog

git reflog

What it does: Shows a log of where HEAD has been, very useful for recovering lost commits.

git blame

git blame file.txt

What it does: Shows who last modified each line of a file.

git fsck

git fsck --full

What it does: Verifies the connectivity and validity of the objects in the database.

git check-ref-format

git check-ref-format --normalize HEAD

What it does: Checks if a reference name (like a branch name) is valid.

git remote show origin

git remote show origin

What it does: Displays detailed information about the 'origin' remote, including its URLs and branches.

git branch -vv

git branch -vv

What it does: Lists local branches with their upstream information and last commit message.

gitk

gitk --all

What it does: A graphical interface for viewing repository history.

git reset --hard HEAD

git reset --hard HEAD

What it does: Discards all changes in the working directory and staging area, returning to the last commit.

git clean -fdx

git clean -fdx

What it does: Removes untracked files and directories, including ignored ones (use with extreme caution).

git stash list

git stash list

What it does: Lists all stashed changes.

git diff --staged

git diff --staged

What it does: Shows changes that are currently in the staging area.

git diff --name-only

git diff --name-only

What it does: Shows only the names of modified files.

git log -S "keyword"

git log -S "my_function_name"

What it does: Finds commits that introduce or remove an instance of a string (pickaxe search).

git log -p <file>

git log -p my_file.txt

What it does: Shows commit history with diffs for a specific file.

git check-ignore -v <file>

git check-ignore -v .env

What it does: Debugs `.gitignore` rules by showing which rule ignored a file.

git rev-parse HEAD

git rev-parse HEAD

What it does: Displays the full hash of the current commit.

git config --show-origin <key>

git config --show-origin user.name

What it does: Shows the value of a configuration key and where it was defined (global, local, etc.).
