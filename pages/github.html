Git Commands

Repository

git init

git init

What it does: Creates a new Git repository in the current directory.

When to use it: To start a new project with Git.

git clone

git clone [URL]

What it does: Clones a repository into a new directory.

When to use it: To get a local copy of a remote repository.

git add

git add .

What it does: Stages changes for the next commit. `.` stages all changes in the current directory.

When to use it: After modifying files and before committing them.

git commit

git commit -m "Your commit message"

What it does: Records a snapshot of the staged changes to the project's history.

When to use it: After staging changes with `git add`.

Branching

git branch

git branch

What it does: Lists all local branches. Use `git branch -a` to see remote branches too.

When to use it: To see the branches in your repository.

git checkout -b

git checkout -b [new-branch-name]

What it does: Creates a new branch and switches to it.

When to use it: To start working on a new feature or bug fix.

Remote

git pull

git pull

What it does: Fetches changes from the remote repository and merges them into the current branch.

When to use it: To update your local repository with the latest changes.

git push

git push origin [branch-name]

What it does: Pushes your committed changes to a remote repository.

When to use it: To share your work with others.

Configuration

git config --global user.name

git config --global user.name "Your Name"

What it does: Sets the name that will be associated with your commits.

When to use it: When setting up Git for the first time.

Troubleshooting

git status

git status

What it does: Displays the state of the working directory and the staging area.

When to use it: A fundamental command to see which files are modified, staged, or untracked.

git log

git log --oneline --graph --decorate

What it does: Shows the commit history.

When to use it: To review the history of commits for a branch.
