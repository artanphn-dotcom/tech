File and Directory Management

ls

ls -lath

What it does: Lists directory contents in a long format, including hidden files, with human-readable sizes, sorted by modification time.

cd

cd /var/log

What it does: Changes the current directory.

pwd

pwd

What it does: Prints the current working directory.

cp

cp -r /source /destination

What it does: Copies files and directories recursively.

mv

mv oldfile.txt newfile.txt

What it does: Moves or renames files and directories.

rm

rm -rf /path/to/dir

What it does: Removes files and directories forcefully and recursively (use with extreme caution).

mkdir

mkdir -p /new/nested/dir

What it does: Creates a new directory, including parent directories if they don't exist.

touch

touch newfile.txt

What it does: Creates an empty file or updates the modification timestamp of an existing file.

find

find . -type f -name "*.log"

What it does: Searches for files based on various criteria like name, type, size, etc.

grep

grep -r "error" /var/log

What it does: Searches for patterns in text recursively within a directory.

cat

cat /etc/passwd

What it does: Concatenates and displays the content of files.

less

less /var/log/syslog

What it does: Views file content one page at a time, allowing for navigation.

head

head -n 20 file.txt

What it does: Displays the first few lines of a file.

tail

tail -f /var/log/syslog

What it does: Displays the last few lines of a file, with `-f` to follow the file in real-time.

chmod

chmod +x script.sh

What it does: Changes the permissions of a file or directory.

chown

chown user:group file.txt

What it does: Changes the owner and group of a file or directory.

ln

ln -s /path/to/original /path/to/link

What it does: Creates a symbolic link to a file or directory.

du

du -sh /var/log

What it does: Shows the disk usage of a directory in a human-readable summary.

df

df -h

What it does: Shows the disk free space on all mounted filesystems.

tar

tar -czvf archive.tar.gz /path/to/dir

What it does: Creates a compressed archive of a directory.

System Information

uname

uname -a

What it does: Displays all system information, including kernel version, hostname, and architecture.

hostname

hostname -I

What it does: Shows the IP addresses of the system.

uptime

uptime

What it does: Shows how long the system has been running and the current load average.

lscpu

lscpu

What it does: Displays detailed information about the CPU architecture.

lsblk

lsblk

What it does: Lists block devices like hard drives and their partitions.

lspci

lspci -nn

What it does: Lists all PCI devices and their vendor/device codes.

lsusb

lsusb

What it does: Lists all USB devices.

free

free -h

What it does: Shows the amount of free and used memory in a human-readable format.

dmesg

dmesg -T

What it does: Prints the kernel ring buffer, useful for diagnosing hardware and driver issues.

who

who -a

What it does: Shows who is logged on to the system.

last

last

What it does: Shows a history of the last logged-in users.

date

date

What it does: Displays the current system date and time.

timedatectl

timedatectl status

What it does: Queries and changes the system clock and its settings.

lshw

sudo lshw -short

What it does: Lists hardware configuration in a summary format.

dmidecode

sudo dmidecode -t system

What it does: Dumps DMI (SMBIOS) data, which contains detailed hardware information.

fdisk

sudo fdisk -l

What it does: Lists disk partition tables.

mount

mount | column -t

What it does: Displays all mounted filesystems in a clean, columnized format.

sysctl

sysctl -a

What it does: Reads or modifies kernel parameters at runtime.

lsof

sudo lsof -n -i

What it does: Lists open files, including network connections.

journalctl

journalctl -b

What it does: Shows all log messages from the current boot.

Networking

ip

ip addr show

What it does: The modern tool for showing and manipulating routing, devices, and tunnels.

ping

ping -c 5 google.com

What it does: Tests network connectivity by sending ICMP packets.

ss

ss -tulnp

What it does: A modern tool for investigating sockets (network connections), showing TCP, UDP, listening, and numeric output with process names.

netstat

netstat -tulnp

What it does: The classic tool for showing network connections, routing tables, and interface statistics.

traceroute

traceroute google.com

What it does: Traces the network path to a destination host.

mtr

mtr google.com

What it does: A powerful combination of `ping` and `traceroute` that provides a continuously updated view of network path performance.

dig

dig +short google.com

What it does: A flexible DNS lookup utility.

nslookup

nslookup google.com

What it does: A simpler DNS lookup utility.

host

host google.com

What it does: A straightforward DNS lookup utility.

curl

curl -I https://google.com

What it does: A versatile tool for transferring data with URLs, often used to test web server responses.

wget

wget https://example.com/file.zip

What it does: A non-interactive tool for downloading files from the internet.

nmap

nmap -sT 192.168.1.1

What it does: A powerful network scanning tool for security audits and port discovery.

tcpdump

sudo tcpdump -i any -n 'port 80'

What it does: A command-line packet analyzer for capturing network traffic.

iptables

sudo iptables -L -n

What it does: The classic tool for configuring the Linux kernel firewall.

ufw

sudo ufw status verbose

What it does: A user-friendly front-end for managing `iptables`.

route

route -n

What it does: A classic tool for showing and manipulating the IP routing table.

arp

arp -n

What it does: Shows the kernel's ARP cache.

ethtool

sudo ethtool eth0

What it does: Queries and controls network driver and hardware settings.

iwconfig

iwconfig

What it does: Configures wireless network interfaces.

ncat

ncat -l -p 8080

What it does: A modern `netcat` for reading and writing data across networks.

ifconfig

ifconfig -a

What it does: Displays network interface configuration (Older tool, `ip addr show` is preferred now but still widely used).

route

route -n

What it does: Shows the IP routing table (Older tool, `ip route show` is preferred now but still widely used).

Process Management

ps

ps aux

What it does: Provides a snapshot of all currently running processes.

top

top

What it does: Provides a real-time, dynamic view of running processes and system resource usage.

htop

htop

What it does: An interactive and more user-friendly version of `top`.

kill

kill -9 12345

What it does: Sends a signal to a process, with `-9` being a forceful termination.

pkill

pkill -f nginx

What it does: Kills processes based on their name or other attributes.

killall

killall firefox

What it does: Kills all processes with a specific name.

systemctl

systemctl status nginx

What it does: The primary tool for managing `systemd` services.

service

service ssh status

What it does: A high-level script for managing System V init services.

lsof

sudo lsof -i :22

What it does: Lists open files, often used to see which process is using a specific port.

pstree

pstree -p

What it does: Displays a tree of running processes showing parent-child relationships.

pgrep

pgrep -u root nginx

What it does: Finds process IDs based on name and other attributes.

pidof

pidof bash

What it does: Finds the process ID of a running program.

jobs

jobs

What it does: Lists jobs running in the background of the current shell.

bg

bg %1

What it does: Puts a suspended process into the background.

fg

fg %1

What it does: Brings a background process to the foreground.

nohup

nohup ./my-long-running-script.sh &

What it does: Allows a process to continue running even after the user has logged out.

nice

nice -n 10 ./my-script.sh

What it does: Runs a command with a modified scheduling priority.

renice

renice 10 -p 12345

What it does: Changes the scheduling priority of a running process.

strace

strace -p 12345

What it does: Traces system calls and signals for a process, useful for deep debugging.

watch

watch -n 2 'ps aux | grep nginx'

What it does: Executes a command repeatedly, showing its output in fullscreen.

Configuration

adduser

sudo adduser newuser

What it does: A user-friendly interactive script to add a new user.

useradd

sudo useradd -m -s /bin/bash newuser

What it does: A low-level utility for adding users.

usermod

sudo usermod -aG sudo newuser

What it does: Modifies a user account, such as adding them to a group.

groupadd

sudo groupadd newgroup

What it does: Creates a new user group.

passwd

passwd

What it does: Changes the password for a user account.

crontab

crontab -e

What it does: Manages cron jobs, which are scheduled tasks.

apt/yum/dnf

sudo apt update && sudo apt upgrade

What it does: Manages packages on Debian-based (apt) or RHEL-based (yum/dnf) systems.

systemctl enable

sudo systemctl enable nginx.service

What it does: Configures a service to start automatically on boot.

sysctl (config)

sudo sysctl -w net.ipv4.ip_forward=1

What it does: Modifies a kernel parameter at runtime. Changes are not persistent across reboots unless added to `/etc/sysctl.conf`.

update-alternatives

sudo update-alternatives --config java

What it does: Manages multiple versions of the same command or application.

hostnamectl

sudo hostnamectl set-hostname new-hostname

What it does: Sets the system hostname.

localectl

sudo localectl set-locale LANG=en_US.UTF-8

What it does: Manages system locale settings.

visudo

sudo visudo

What it does: Safely edits the `/etc/sudoers` file to configure user privileges.

fstab

# Edit /etc/fstab to manage persistent mounts

What it does: The filesystem table defines how disk partitions and storage devices are mounted.

bashrc / profile

# Edit ~/.bashrc or ~/.profile

What it does: Configures the shell environment for a user, such as aliases and environment variables.

ssh-keygen

ssh-keygen -t rsa -b 4096

What it does: Generates a new SSH key pair for secure authentication.

ufw allow

sudo ufw allow ssh

What it does: A simple way to configure firewall rules.

grub-mkconfig

sudo grub-mkconfig -o /boot/grub/grub.cfg

What it does: Generates a new GRUB boot loader configuration file.

modprobe

sudo modprobe a-kernel-module

What it does: Adds or removes kernel modules.

ip route add

sudo ip route add default via 192.168.1.1

What it does: Adds a new route to the routing table.

Troubleshooting

dmesg

dmesg -T | grep -i error

What it does: Displays kernel ring buffer messages, filtered for errors.

journalctl

journalctl -u nginx.service -f

What it does: Follows the logs for a specific systemd service in real-time.

tcpdump

sudo tcpdump -i any -n 'port 443'

What it does: Captures and analyzes network packets, essential for deep network troubleshooting.

lsof

sudo lsof -i :80

What it does: Lists open files and can identify which process is listening on a specific port.

strace

sudo strace -p $(pidof nginx)

What it does: Traces system calls made by a process, useful for debugging application issues.

iostat

iostat -dx 2

What it does: Reports CPU and I/O statistics, refreshing every 2 seconds.

vmstat

vmstat 2 5

What it does: Reports virtual memory statistics, running 5 times with a 2-second interval.

mpstat

mpstat -P ALL 1

What it does: Shows CPU statistics for all processors, refreshing every second.

sar

sar -u 1 5

What it does: Collects and reports system activity information, in this case CPU usage.

iotop

sudo iotop

What it does: An interactive tool to monitor I/O usage by processes.

iftop

sudo iftop -i eth0

What it does: An interactive tool to monitor network bandwidth usage by connections.

nethogs

sudo nethogs eth0

What it does: An interactive tool that groups bandwidth usage by process.

nc

nc -zv 192.168.1.100 22

What it does: A simple tool to test connectivity to a specific TCP/UDP port.

openssl s_client

openssl s_client -connect google.com:443

What it does: A client for testing SSL/TLS connections and viewing certificate details.

mtr

mtr --report google.com

What it does: Combines ping and traceroute to diagnose network path issues and packet loss.

ss -s

ss -s

What it does: Provides a summary of all socket statistics.

ip route get

ip route get 8.8.8.8

What it does: Shows which route will be taken for a specific destination IP.

systemd-analyze

systemd-analyze blame

What it does: Shows which systemd services are taking the most time during boot.

findmnt

findmnt /path/to/dir

What it does: Finds which filesystem a directory belongs to.

stat

stat /path/to/file.txt

What it does: Displays detailed status information for a file or filesystem.
