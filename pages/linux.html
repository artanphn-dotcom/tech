File and Directory Management

ls - List directory contents

ls -la

What it does: Lists files and directories with detailed information.

When to use it: For navigating the filesystem and inspecting file permissions and metadata.

cd - Change directory

cd /var/log

What it does: Navigates to the specified directory.

When to use it: To move between directories.

pwd - Print working directory

pwd

What it does: Displays the full path of the current directory.

When to use it: When you need to know the absolute path of your current location.

cp - Copy files

cp source.txt destination.txt

What it does: Copies files or directories.

When to use it: To duplicate files. Use `cp -r` for directories.

mv - Move or rename files

mv old-name.txt new-name.txt

What it does: Moves or renames files and directories.

When to use it: To relocate files or change their names.

rm - Remove files

rm file.txt

What it does: Deletes files. Use with caution.

When to use it: To permanently delete files. Use `rm -r` for directories.

find - Search for files

find . -name "*.txt"

What it does: Searches for files in a directory hierarchy.

When to use it: To locate files based on name, size, modification time, etc.

grep - Search text

grep "error" /var/log/syslog

What it does: Searches for a pattern of text within files.

When to use it: Essential for finding specific lines in log files or any text document.

System Information

uname - Kernel information

uname -a

What it does: Prints detailed information about the Linux kernel and system architecture.

When to use it: To quickly identify the kernel version, hostname, and processor type.

df - Disk space usage

df -h

What it does: Reports the amount of available disk space on all mounted filesystems.

When to use it: To monitor disk capacity. The `-h` flag makes the output human-readable.

free - Memory usage

free -h

What it does: Shows the amount of free and used memory in the system.

When to use it: To check memory consumption and availability.

Networking

ip addr show - Display IP addresses

ip addr show

What it does: Shows all network interfaces and their assigned IP addresses.

When to use it: For network troubleshooting and verifying IP configurations.

ping - Check network connectivity

ping 8.8.8.8

What it does: Sends ICMP packets to a host to test connectivity.

When to use it: To check if a remote host is reachable.

netstat - Network statistics

netstat -tuln

What it does: Displays network connections, routing tables, and interface statistics.

When to use it: To see which ports are open and what connections are active.

ss - Socket statistics

ss -tuln

What it does: A modern replacement for netstat, used to investigate sockets.

When to use it: Offers better performance than netstat for viewing network connections.

Process Management

ps - Process status

ps aux

What it does: Provides a snapshot of all currently running processes.

When to use it: To see which processes are running and inspect their details.

top - Display running processes

top

What it does: Provides a real-time, dynamic view of the processes running on the system.

When to use it: To monitor system resource usage (CPU, memory) in real time.

kill - Terminate a process

kill 12345

What it does: Sends a signal to a process, typically to terminate it.

When to use it: To stop a misbehaving or unresponsive process. Use `kill -9` for a force kill.

Configuration

ip addr add - Set IP address

sudo ip addr add 192.168.1.10/24 dev eth0

What it does: Assigns an IP address and subnet mask to a network interface.

When to use it: To manually configure a static IP address on a server.

iptables - Firewall configuration

sudo iptables -A INPUT -s 10.0.0.5 -j DROP

What it does: Configures the netfilter firewall. This example blocks all incoming traffic from a specific IP.

When to use it: To set up basic firewall rules. Note that `nftables` is replacing `iptables` on many modern systems.

Troubleshooting

tcpdump - Packet sniffer

sudo tcpdump -i eth0 -n 'port 80'

What it does: Captures and analyzes network packets in real time.

When to use it: Essential for deep network troubleshooting to see exactly what traffic is flowing.

lsof - List open files

sudo lsof -i :443

What it does: Lists all open files and the processes that opened them. Can be used to see which process is using a specific port.

When to use it: To identify which process is listening on a certain network port.

journalctl - Query the systemd journal

journalctl -u nginx.service -f

What it does: Displays logs from the systemd journal. This example follows the logs for the nginx service.

When to use it: The standard way to view system and service logs on modern Linux distributions.
