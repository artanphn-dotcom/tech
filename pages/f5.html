System

show /sys version

show /sys version

show /sys uptime

show /sys uptime

What it does: Shows system uptime and load average.

show /sys hardware

show /sys hardware

What it does: Displays detailed hardware information for the BIG-IP device.

show /sys ha-status

show /sys ha-status

What it does: Shows High Availability (HA) status of the device, including active/standby state.

show /sys performance system

show /sys performance system

What it does: Displays system-wide performance statistics (CPU, memory, disk I/O).

show /sys memory

show /sys memory

What it does: Shows memory usage breakdown.

show /sys disk directory

show /sys disk directory

What it does: Displays disk usage for various partitions.

show /sys log all

show /sys log all

What it does: Displays all system logs (equivalent to `tail -f /var/log/ltm`).

show /sys license

show /sys license

What it does: Shows installed license details.

show /sys connection

show /sys connection

What it does: Displays the full connection table, showing active connections processed by BIG-IP.

show /net interface

show /net interface

What it does: Displays status and statistics for all network interfaces.

show /net self

show /net self

What it does: Shows configured self-IP addresses.

show /net route

show /net route

What it does: Displays the BIG-IP routing table.

show /auth user

show /auth user

What it does: Lists configured local user accounts.

show /cm device

show /cm device

What it does: Shows status of all devices in the Device Service Clustering (DSC) group.

show /cm sync-status

show /cm sync-status

What it does: Displays the configuration synchronization status for HA/DSC.

show /security firewall policy

show /security firewall policy

What it does: Displays configured firewall policies.

show /apm policy access-profile

show /apm policy access-profile

What it does: Lists Access Policy Manager (APM) access profiles.

show /ltm profile client-ssl

show /ltm profile client-ssl

What it does: Displays configured client SSL profiles.

show /ltm profile server-ssl

show /ltm profile server-ssl

What it does: Displays configured server SSL profiles.

Virtual Server

show /ltm virtual

show /ltm virtual

What it does: Displays the status and configuration of all virtual servers.

show /ltm virtual <vs-name>

show /ltm virtual my_vs

What it does: Shows detailed status and configuration for a specific virtual server.

show /ltm virtual <vs-name> stats

show /ltm virtual my_vs stats

What it does: Displays detailed statistics for a specific virtual server.

modify /ltm virtual <vs-name> disabled

modify /ltm virtual my_vs disabled

What it does: Disables a virtual server, stopping it from processing new connections.

modify /ltm virtual <vs-name> enabled

modify /ltm virtual my_vs enabled

What it does: Enables a previously disabled virtual server.

create /ltm virtual <vs-name> destination <ip:port> pool <pool-name>

create /ltm virtual my_vs destination 10.1.1.10:80 pool my_pool

What it does: Creates a basic virtual server. This is an example; full config is complex.

delete /ltm virtual <vs-name>

delete /ltm virtual my_vs

What it does: Deletes a virtual server (use with caution).

show /ltm virtual <vs-name> persistence

show /ltm virtual my_vs persistence

What it does: Displays persistence entries for a virtual server.

show /ltm virtual <vs-name> clientssl-profile

show /ltm virtual my_vs clientssl-profile

What it does: Shows the client SSL profile applied to a virtual server.

show /ltm virtual <vs-name> serverssl-profile

show /ltm virtual my_vs serverssl-profile

What it does: Shows the server SSL profile applied to a virtual server.

show /ltm virtual <vs-name> rules

show /ltm virtual my_vs rules

What it does: Displays iRules applied to a virtual server.

show /ltm virtual <vs-name> policies

show /ltm virtual my_vs policies

What it does: Shows LTM policies applied to a virtual server.

show /ltm virtual <vs-name> profiles

show /ltm virtual my_vs profiles

What it does: Displays all profiles (e.g., HTTP, TCP, SSL) applied to a virtual server.

show /ltm virtual <vs-name> translate-address

show /ltm virtual my_vs translate-address

What it does: Shows if address translation is enabled for the virtual server.

show /ltm virtual <vs-name> translate-port

show /ltm virtual my_vs translate-port

What it does: Shows if port translation is enabled for the virtual server.

show /ltm virtual <vs-name> address-status

show /ltm virtual my_vs address-status

What it does: Displays the status of the virtual server's IP address (e.g., floating/non-floating).

show /ltm virtual <vs-name> connection-mirror

show /ltm virtual my_vs connection-mirror

What it does: Shows if connection mirroring is enabled for the virtual server in an HA setup.

show /ltm virtual <vs-name> vs-index

show /ltm virtual my_vs vs-index

What it does: Displays the virtual server's index number, useful for certain debug commands.

show /ltm virtual <vs-name> destination

show /ltm virtual my_vs destination

What it does: Shows the destination IP and port the virtual server listens on.

show /ltm virtual <vs-name> source-address-translation

show /ltm virtual my_vs source-address-translation

What it does: Displays the source address translation (SNAT) configuration for the virtual server.

Pool

show /ltm pool

show /ltm pool

What it does: Displays the status of all pools of backend servers.

show /ltm pool <pool-name>

show /ltm pool my_pool

What it does: Shows detailed status and configuration for a specific pool.

show /ltm pool <pool-name> members

show /ltm pool my_pool members

What it does: Displays the status of all members (nodes) within a specific pool.

show /ltm pool <pool-name> stats

show /ltm pool my_pool stats

What it does: Displays detailed statistics for a specific pool.

modify /ltm pool <pool-name> monitor <monitor-name>

modify /ltm pool my_pool monitor http_monitor

What it does: Assigns a health monitor to a pool.

modify /ltm pool <pool-name> members modify { address <ip> port <port> { session | state } disabled }

modify /ltm pool my_pool members modify { 10.0.0.1:80 { session user-disabled state user-down } }

What it does: Disables a specific pool member's session and marks it down (graceful shutdown).

modify /ltm pool <pool-name> members modify { address <ip> port <port> { session | state } enabled }

modify /ltm pool my_pool members modify { 10.0.0.1:80 { session enabled state enabled } }

What it does: Enables a specific pool member.

create /ltm pool <pool-name> members add { <ip:port> }

create /ltm pool my_new_pool members add { 10.0.0.100:80 }

What it does: Creates a new pool and adds an initial member.

delete /ltm pool <pool-name>

delete /ltm pool my_pool

What it does: Deletes a pool (use with caution).

show /ltm pool <pool-name> load-balancing-mode

show /ltm pool my_pool load-balancing-mode

What it does: Displays the load balancing method configured for a pool.

show /ltm pool <pool-name> min-active-members

show /ltm pool my_pool min-active-members

What it does: Shows the minimum number of active members required for the pool to be up.

show /ltm monitor

show /ltm monitor

What it does: Lists all configured health monitors.

show /ltm monitor <monitor-name>

show /ltm monitor http_monitor

What it does: Displays configuration details for a specific health monitor.

show /ltm pool <pool-name> members field-fmt

show /ltm pool my_pool members field-fmt

What it does: Displays pool member details in a more structured, parsable format.

show /ltm pool <pool-name> priority-group

show /ltm pool my_pool priority-group

What it does: Shows the priority group activation setting for a pool.

show /ltm pool <pool-name> session

show /ltm pool my_pool session

What it does: Displays the session status (enabled/disabled) for the pool itself.

show /ltm pool <pool-name> state

show /ltm pool my_pool state

What it does: Shows the operational state (up/down) for the pool itself.

show /ltm pool <pool-name> members | grep "down"

show /ltm pool my_pool members | grep "down"

What it does: Filters pool members to show only those marked as 'down'.

show /ltm pool <pool-name> description

show /ltm pool my_pool description

What it does: Displays the description configured for a pool.

show /ltm pool <pool-name> all-properties

show /ltm pool my_pool all-properties

What it does: Shows all configured properties for a pool.

Node

show /ltm node

show /ltm node

What it does: Displays the status of all nodes (backend servers).

show /ltm node <node-name>

show /ltm node 10.0.0.10

What it does: Shows detailed status and configuration for a specific node.

show /ltm node <node-name> stats

show /ltm node 10.0.0.10 stats

What it does: Displays detailed statistics for a specific node.

modify /ltm node <node-name> session user-disabled

modify /ltm node 10.0.0.10 session user-disabled

What it does: Disables a node, preventing new connections from being sent to it (graceful shutdown).

modify /ltm node <node-name> session enabled

modify /ltm node 10.0.0.10 session enabled

What it does: Enables sessions for a node, allowing it to receive new connections.

modify /ltm node <node-name> state user-down

modify /ltm node 10.0.0.10 state user-down

What it does: Forces a node to a "down" state, removing it from all active pools.

modify /ltm node <node-name> state enabled

modify /ltm node 10.0.0.10 state enabled

What it does: Forces a node to an "up" state, re-adding it to pools.

create /ltm node <node-name> address <ip>

create /ltm node my_web_server address 10.0.0.10

What it does: Creates a new node object.

delete /ltm node <node-name>

delete /ltm node my_web_server

What it does: Deletes a node object (use with caution).

show /ltm node <node-name> monitor-status

show /ltm node 10.0.0.10 monitor-status

What it does: Displays the health monitor status for a specific node.

show /ltm node <node-name> connection-limit

show /ltm node 10.0.0.10 connection-limit

What it does: Shows the maximum number of connections allowed for a node.

show /ltm node <node-name> rate-limit

show /ltm node 10.0.0.10 rate-limit

What it does: Displays the connection rate limit for a node.

show /ltm node <node-name> availability-state

show /ltm node 10.0.0.10 availability-state

What it does: Shows the overall availability state (e.g., available, unknown, offline).

show /ltm node <node-name> status

show /ltm node 10.0.0.10 status

What it does: Shows the detailed status of a node, combining availability and enabled states.

show /ltm node <node-name> dynamically-discovered

show /ltm node 10.0.0.10 dynamically-discovered

What it does: Indicates if a node was dynamically discovered (e.g., by auto-populate on a pool).

show /ltm node <node-name> description

show /ltm node 10.0.0.10 description

What it does: Displays the description configured for a node.

show /ltm node <node-name> all-properties

show /ltm node 10.0.0.10 all-properties

What it does: Shows all configured properties for a node.

show /ltm node <node-name> address

show /ltm node 10.0.0.10 address

What it does: Displays the IP address associated with the node.

show /ltm node <node-name> port

show /ltm node 10.0.0.10 port

What it does: Displays the default service port associated with the node.

show /ltm node <node-name> monitor

show /ltm node 10.0.0.10 monitor

What it does: Shows the health monitors explicitly applied to the node (if any).

Configuration

create /ltm pool <pool-name> members add { <ip:port> } monitor <monitor-type>

create /ltm pool my-pool members add { 10.0.0.1:80 10.0.0.2:80 } monitor http

What it does: Creates a new load balancing pool with members and a health monitor.

create /ltm virtual <vs-name> destination <ip:port> pool <pool-name>

create /ltm virtual my-virtual-server destination 192.168.1.100:80 ip-protocol tcp pool my-pool

What it does: Creates a new virtual server that listens for traffic and sends it to the specified pool.

load sys config merge from-terminal

load sys config merge from-terminal

What it does: Loads configuration changes typed directly into the terminal, merging them with the running config.

save sys config

save sys config

What it does: Saves the current running configuration to disk.

tmsh create <module> <object-type> <name>

tmsh create /ltm node my_new_node address 10.0.0.150

What it does: General syntax for creating objects using TMSH.

tmsh modify <module> <object-type> <name>

tmsh modify /ltm virtual my_vs profiles add { http { } }

What it does: General syntax for modifying existing objects.

tmsh delete <module> <object-type> <name>

tmsh delete /ltm node my_old_node

What it does: General syntax for deleting objects.

install sys software image <image-name> volume <volume-name>

install sys software image BIGIP-16.1.0-0.0.1.iso volume HD1.2

What it does: Installs a BIG-IP software image to a specified boot volume.

create /sys management-ip <ip/mask>

create /sys management-ip 192.168.1.100/24

What it does: Configures the management IP address.

create /sys global-settings hostname <hostname>

create /sys global-settings hostname my-f5-lb

What it does: Sets the hostname of the BIG-IP device.

create /sys ntp servers add { <ntp-ip> }

create /sys ntp servers add { 1.pool.ntp.org }

What it does: Configures NTP servers for time synchronization.

create /ltm monitor http <monitor-name> send "GET /" recv "HTTP/1.1 200 OK"

create /ltm monitor http my_http_monitor send "GET /" recv "HTTP/1.1 200 OK"

What it does: Creates a custom HTTP health monitor.

create /ltm persistence source-addr <profile-name> defaults-from source_addr

create /ltm persistence source-addr my_source_addr_persist defaults-from source_addr

What it does: Creates a new source address affinity persistence profile.

create /ltm rule <rule-name> definition { when HTTP_REQUEST { log local0. "HTTP Request from [IP::client_addr]" } }

create /ltm rule my_irule definition { when HTTP_REQUEST { log local0. "HTTP Request from [IP::client_addr]" } }

What it does: Creates a simple iRule.

create /sys folder <folder-name>

create /sys folder my_application_folder

What it does: Creates a new administrative folder for organizing objects.

create /sys crypto key generate <key-type> <name>

create /sys crypto key generate rsa my_new_key

What it does: Generates a new cryptographic key.

create /sys crypto cert generate <name> country <country>

create /sys crypto cert generate my_new_cert country US

What it does: Generates a new self-signed certificate.

create /ltm profile one-connect <profile-name> defaults-from oneconnect

create /ltm profile one-connect my_oneconnect_profile defaults-from oneconnect

What it does: Creates a OneConnect profile for connection reuse.

create /ltm profile http <profile-name> defaults-from http

create /ltm profile http my_http_profile defaults-from http insert-xforwarded-for X-Forwarded-For

What it does: Creates a custom HTTP profile with X-Forwarded-For header insertion.

create /ltm profile tcp <profile-name> defaults-from tcp

create /ltm profile tcp my_tcp_profile defaults-from tcp

What it does: Creates a custom TCP profile.

Troubleshooting

show /sys connection

show /sys connection cs-server-addr 192.168.1.100

What it does: Displays detailed information about current connections, filtered by virtual server IP.

tail -f /var/log/ltm

show /sys performance top

show /sys performance top

What it does: Shows top resource-consuming processes and statistics.

run /util ping <ip-address>

run /util ping 8.8.8.8

What it does: Performs a basic ping test from the BIG-IP device.

run /util traceroute <ip-address>

run /util traceroute 8.8.8.8

What it does: Performs a traceroute from the BIG-IP device to diagnose network path issues.

run /util tcpdump -i <interface> host <ip>

run /util tcpdump -i 1.1 host 10.0.0.10

What it does: Captures network traffic on a specific interface, filtered by host.

show /sys log apm

show /sys failover

show /sys failover

show /sys application service

show /sys application service

show /sys ip-address

show /sys cluster all
