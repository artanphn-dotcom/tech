Virtual Server

show /ltm virtual

show /ltm virtual

What it does: Displays the status and configuration of all virtual servers.

When to use it: To get a quick overview of your virtual servers and their current state.

Pool

show /ltm pool

show /ltm pool

What it does: Displays the status of all pools of backend servers.

When to use it: To check the status of pools and their members.

Node

show /ltm node

show /ltm node

What it does: Displays the status of all nodes (backend servers).

When to use it: To check the health and status of individual backend servers.

modify /ltm node [node_ip] session user-disabled

modify /ltm node 10.0.0.10 session user-disabled

What it does: Disables a node, preventing new connections from being sent to it.

When to use it: To gracefully take a server out of rotation for maintenance.

Configuration

create /ltm pool

create /ltm pool my-pool members add { 10.0.0.1:80 10.0.0.2:80 } monitor http

What it does: Creates a new load balancing pool with two members and an HTTP health monitor.

When to use it: When setting up a new application or service.

create /ltm virtual

create /ltm virtual my-virtual-server destination 192.168.1.100:80 ip-protocol tcp pool my-pool

What it does: Creates a new virtual server that listens for traffic and sends it to the specified pool.

When to use it: The final step in exposing a new load-balanced service.

Troubleshooting

show /sys connection

show /sys connection cs-server-addr [virtual_server_ip]

What it does: Displays detailed information about current connections to a virtual server.

When to use it: To troubleshoot live traffic flows and inspect connection details.
