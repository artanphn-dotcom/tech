System

show system info

show system info

What it does: Displays system version, uptime, serial, and hostname.

When to use it: For quickly identifying the device and its current software state.

show clock

show clock

What it does: Displays the current date and time of the firewall.

When to use it: To verify the system clock, which is important for logging and certificate validation.

Interface

show interface all

show interface all

What it does: Verifies interface status, speed, and link state.

When to use it: To check the link status, IP addresses, and counters for all interfaces.

show interface ethernet1/1

show interface ethernet1/1

What it does: Displays detailed information for a specific interface.

When to use it: To troubleshoot issues on a single interface, such as errors or traffic counters.

Routing

show routing route

show routing route

What it does: Shows routing table and route selection.

When to use it: To inspect the firewall's routing table and verify route paths.

test routing fib-lookup ip <destination>

test routing fib-lookup ip 8.8.8.8

What it does: Confirms which route/interface will be used for a specific destination IP.

When to use it: To troubleshoot routing decisions and verify next-hop reachability.

VPN

show vpn ike-sa

show vpn ike-sa

What it does: Checks IKE Phase 1 status for VPNs.

When to use it: To verify if the IKE security association (Phase 1) is established.

show vpn ipsec-sa

show vpn ipsec-sa

What it does: Verifies IPsec Phase 2 tunnel status and counters.

When to use it: To check if the IPsec security association (Phase 2) is established and actively passing traffic.

test vpn ike-sa gateway <name>

test vpn ike-sa gateway my-vpn-gw

What it does: Initiates or tests an IKE VPN tunnel.

When to use it: To manually bring up a VPN tunnel or test its configuration.

Security Policy

show running security-policy

show running security-policy

What it does: Displays all configured security policy rules.

When to use it: To review and verify the firewall's security rulebase.

test security-policy-match

test security-policy-match from [zone] to [zone] source [ip] destination [ip] protocol [number]

What it does: Simulates a packet flow to see which security policy it would match.

When to use it: Extremely useful for troubleshooting why traffic is being allowed or denied.

Configuration Mode

configure

configure

What it does: Enters the device's configuration mode.

When to use it: All configuration changes must be made in this mode.

set deviceconfig system hostname

set deviceconfig system hostname my-firewall

What it does: Sets the hostname of the firewall.

When to use it: A basic configuration step to identify the device on the network.

commit

commit

What it does: Validates and applies the candidate configuration to the running configuration.

When to use it: After making configuration changes, you must commit them to make them active.

Troubleshooting

show session all filter source <ip>

show session all filter source 10.1.1.1

What it does: Troubleshoots traffic sessions and policies by displaying active firewall sessions filtered by source IP.

When to use it: A primary tool for troubleshooting live traffic flows.

ping

ping source [ip] host [destination_ip]

What it does: Sends ICMP packets to test reachability.

When to use it: To test basic network connectivity from the firewall itself.

debug dataplane packet-diag set filter match <criteria>

debug dataplane packet-diag set filter match source 10.1.1.1 destination 8.8.8.8
debug dataplane packet-diag set log feature flow basic
debug dataplane packet-diag on

What it does: Captures traffic for deep packet troubleshooting. A powerful packet capture and debugging tool for the dataplane.

When to use it: For in-depth analysis of how a packet is being processed by the firewall.

less mp-log system.log

less mp-log system.log

What it does: Reads system logs for errors and events.

When to use it: To review system-level events and troubleshoot appliance issues.
